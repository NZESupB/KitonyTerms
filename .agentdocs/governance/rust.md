# Rust 开发治理

## 项目形态
- 当前项目是 Rust workspace，主要 crate 包括 `kt-core`、`kt-config`、`kt-secrets`、`kt-ui`、`kt-app`。
- 修改前应先确认影响范围，优先复用现有模块与类型，不引入额外框架或依赖，除非能显著降低安全风险或复杂度。

## 验证要求
- Rust 代码变更后必须运行 `cargo fmt`。
- Rust 代码变更后必须运行 `cargo test`。
- 涉及安全边界、网络、认证、密钥、文件系统路径、外部命令或并发状态时，应运行 `cargo clippy --all-targets --all-features`，除非本地环境无法完成并已记录原因。

## 安全基线
- 不记录、展示或持久化明文密码、私钥内容、一次性验证码、会话令牌等敏感信息。
- 文件路径、远端路径、外部命令参数与配置输入都应有明确边界，避免路径穿越、命令注入、权限过宽或静默降级。
- 处理 SSH host key、认证失败、代理连接和 SFTP 操作时，应让错误可见且可恢复，不应在失败后继续执行危险操作。
